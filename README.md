## 📂 Структура проекта

```text
├── README.md              — описание проекта
├── test-plan.md           — тест-план
├── checklists/
│   ├── search.md          — чек-лист поиска
│   ├── cart.md            — чек-лист корзины
│   └── auth.md            — чек-лист регистрации
├── test-cases/
│   ├── search_TC.md       — тест-кейсы поиска
│   ├── cart_TC.md         — тест-кейсы корзины
│   └── auth_TC.md         — тест-кейсы регистрации
├── bug-reports/
│   └── bugs.md            — найденные баги
└── sql/
    └── queries.md         — SQL запросы
```

### Зачем разделены папки?
- **`checklists/`**: Созданы для проведения быстрой exploratory (исследовательской) проверки функционала.
- **`test-cases/`**: Содержат детальные сценарии, которые формализуют процесс тестирования и помогают при регрессе.
- **`bug-reports/`**: Используются для фиксации дефектов, чтобы обеспечить их прозрачность и дальнейшую проработку.
- **`sql/`**: Предназначены для демонстрации навыков проверки данных на уровне БД (верификация запросов).
